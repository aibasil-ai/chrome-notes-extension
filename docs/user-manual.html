1. 快速開始（商店安裝版）

從 Chrome 線上應用程式商店安裝

  1. 前往 Chrome 線上應用程式商店。
  2. 搜尋 輕筆記，進入擴充功能頁面後按「加到 Chrome」。
  3. 在權限視窗確認後按「新增擴充功能」。
  4. 安裝完成後，點擊瀏覽器右上角「擴充功能」圖示，將輕筆記釘選到工具列。
  5. 點一下輕筆記圖示，開啟 Popup 並建立第一則筆記。

首次啟用建議檢查

  1. 開啟設定，確認「同步容量滿載時允許僅儲存本機」是否符合你的策略。
  2. 確認自動儲存間隔（預設 3000ms）。
  3. 建立一則測試筆記，確認可正常儲存與搜尋。
透過 Chrome 商店安裝的版本通常由 Chrome 自動更新；若你要手動檢查更新，可到 chrome://extensions/ 按「更新」。

2. 三種介面模式

Popup 模式（快速記錄）

  • 適合快速新增、查看與修改短筆記。
  • 可直接切換到 Sidebar 或獨立視窗。

Sidebar 模式（邊看邊記）

  • 在瀏覽網頁時持續筆記，不中斷原頁面閱讀。
  • 適合研究整理、摘錄重點與即時補充。

獨立視窗模式（深度整理）

  • 提供較大編輯空間，適合長文筆記與多段整理。
  • 可拖曳調整左側筆記清單與右側編輯區寬度。

3. 建立與編輯筆記

建立新筆記

  1. 按「新增筆記」。
  2. 輸入標題、內容與標籤。
  3. 選擇「純文字」或「Markdown」。
  4. 按「儲存」完成。

自動儲存規則

  • 內容變更後，系統依設定間隔自動儲存。
  • 新筆記若標題與內容都為空，不會觸發自動儲存。
  • 遇到容量或大小限制時，會顯示對應提示訊息。

記錄網頁資訊

  • 勾選「記錄網頁」可保存當下分頁網址與標題。
  • 編輯既有筆記時可勾選「儲存時更新為目前分頁網址」。
單筆筆記若超過 7,800 bytes 會被阻擋儲存。建議改成「摘要 + 外部連結」或拆分成多則筆記。

4. 搜尋與標籤管理

  • 搜尋列會同時比對標題與內容關鍵字。
  • 可搭配多標籤過濾，快速縮小範圍。
  • 建議採固定標籤命名（例如：研究、待辦、已驗證）。
使用情境 建議標籤策略
長期專題追蹤 主題標籤 + 階段標籤（例如 專案A + 草稿）
臨時紀錄 先統一標為 inbox，每週整理歸類
可行動任務 加入 todo 或時間標籤（例如 2026-Q2）

5. 同步與容量規則

輕筆記採 chrome.storage.local + chrome.storage.sync 混合策略： 先保護當前資料，再依容量規則嘗試同步。

項目 限制值 說明
單筆同步上限 7,800 bytes 超過就直接阻擋儲存。
同步容量保護線 96 KB (98304 bytes) 超過後依設定決定阻擋或改為僅存本機。
同步筆數防呆上限 512 僅作保護值，避免極端狀況。

「同步容量滿載時允許僅儲存本機」開關差異

  • 開啟：無法同步時仍可存本機，筆記會標示為未同步。
  • 關閉：無法同步時直接阻擋，這次變更不會儲存。
若你高度依賴跨裝置一致性，建議維持「關閉」。若你重視不中斷記錄，可開啟並定期清理同步空間。

6. 匯出與匯入

匯出 JSON

  1. 打開設定面板。
  2. 點擊「匯出 JSON」。
  3. 將檔案保存到雲端硬碟或版本化備份資料夾。

匯入 JSON

  1. 在設定中按「選擇檔案」。
  2. 選取已備份的 JSON 檔。
  3. 系統會合併匯入，已存在 ID 的筆記不重複新增。
若同步空間接近上限，大量匯入可能觸發容量阻擋。建議先匯出再整理舊筆記。

7. 常見問題

Q1. 為什麼筆記會顯示「未同步」？

通常代表該筆目前只存在本機，常見原因是同步空間不足或筆記超過單筆大小上限。

Q2. 關閉「僅儲存本機」後，既有本機筆記會不見嗎？

不會。既有本機筆記會保留；但之後若新變更無法同步，就會直接阻擋儲存。

Q3. 為什麼使用量有時會顯示高於 96KB？

96KB 是應用層保護線，用來提前防止接近 Chrome 原生配額時的不確定行為。

8. 疑難排解

情境：儲存時出現容量提示

  1. 到設定頁檢查同步使用量是否逼近或超過 96KB。
  2. 評估是否暫時開啟「同步容量滿載時允許僅儲存本機」。
  3. 刪除大型舊筆記，或先匯出後再清理。

情境：長筆記常遇到 7,800B 限制

  1. 改為主筆記 + 子筆記結構（摘要、細節、參考資料）。
  2. 大型原文改放外部文件，筆記內保留摘要與連結。

情境：跨裝置看不到最新筆記

  1. 在來源裝置儲存後先等待幾秒。
  2. 在目標裝置開啟擴充功能檢查該筆是否未同步。
  3. 若長時間不同步，回來源裝置檢查容量與錯誤提示。
建議在大量整理前先匯出一份 JSON 備份，能大幅降低誤刪或衝突風險。

下一步：建立你的第一套穩定筆記流程

建議先建立 3 個固定標籤（例如：研究、待辦、已驗證），並在每次長篇整理前先做 JSON 備份。 這能讓搜尋、同步與回顧都更穩定。
